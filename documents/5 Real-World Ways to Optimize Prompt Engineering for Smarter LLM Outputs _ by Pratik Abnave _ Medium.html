5 Real-World Ways to Optimize Prompt Engineering for Smarter LLM Outputs

Pratik Abnave
4 min read
·
May 9, 2025

Here’s a thought that might seem obvious, but isn’t: Most LLMs are pretty good. Our prompts aren’t.

A while ago, I set up a GPT-4-based helper for a legal team to study court decisions and write case summaries. The tech side worked fine. But the output? Useless fluff. Too wordy, basic, and — worst of all — missed the point.

The AI wasn’t to blame. I was.

I was giving vague directions to one of the world’s smartest AI systems — and hoping for genius.

That’s when I got hooked on prompt engineering. Not the quick tricks you see on Twitter. Serious, organized, step-by-step prompt design.

Here are five real-world tips that helped me go from “meh” answers to “wow, that’s clever” — and how you can use them in your own LangChain or open-source LLM setup.

1. Give the Model a Job — It Does Help

Have you noticed how people react when you tell them who they are in a situation?

LLMs work the same way.

I typed in one simple sentence:

“You are a senior legal analyst at a Mumbai-based firm.”

…and it changed everything.

Right away, the responses became more focused. Sharper. On point.

LangChain Prompt Template

from langchain.prompts import PromptTemplate
role_prompt = PromptTemplate(
 input_variables=["role", "task", "focus"],
 template="""
You are a {role}.
Your task is to {task}.
Please focus on the following aspects: {focus}.
Structure your answer in concise bullet points.
"""
)
# Usage
print(role_prompt.format(
 role="senior financial analyst",
 task="review the quarterly earnings call",
 focus="revenue growth, margin pressure, and FY24 guidance"
))

Use this when building financial assistants, legal bots, or any LLM tool that needs contextual authority.

Take It Step by Step (Thought Process)

This approach doesn’t get enough credit. For tasks with multiple steps or that need reasoning (math, logic, grouping, problem-solving), don’t just ask for the answer. Get the model to explain its thinking.

“Let’s work through this one step at a time.”

That simple sentence can turn a quick summary into a logical argument.

LangChain Template

cot_prompt = PromptTemplate(
 input_variables=["question"],
 template="""
Let's work through this step by step.
Question: {question}
"""
)
# Example
cot_prompt.format(question="If a train leaves at 3 PM going 80 km/h, how far does it travel in 4.5 hours?")

Pro Tip: Use this format in technical troubleshooting bots, math tutors, and logic-heavy chat flows.

3. Give Examples:

Use Few-Shot Learning AI models are advanced pattern-finders. Show them what you want before asking them to apply it.

Let’s say you’re creating a tool to sort opinions.

This works great for: Figuring out what people mean Spotting the mood of text Organizing reviews

few_shot_prompt = PromptTemplate(
 input_variables=["input"],
 template="""
Categorize the sentiment:
1. "This app is amazing!" → Positive 
2. "It's fine, I guess." → Neutral 
3. "Crashes every time I open it." → Negative 
4. "{input}" →
"""
)
few_shot_prompt.format(input="Battery life is terrible but the design is nice.")

Works wonders in:

  • Intent classification
  • Tone detection
  • Review sorting

Keep it clean — 3–5 examples are enough. More just eat tokens.

4. Create Flexible Templates with Placeholders

Let’s say you’re building a system to summarize documents. You don’t want to write out every prompt. You need something you can change and use in real products.

Here’s how I’ve done this in a Flask + LangChain setup:

dynamic_template = PromptTemplate(
 input_variables=["role", "doc_type", "tone", "content"],
 template="""
You are a {role}.
Analyze the following {doc_type} in a {tone} tone:
{content}
Summarize the main points in bullet format.
"""
)
# Example
dynamic_template.format(
 role="AI legal advisor",
 doc_type="judicial opinion",
 tone="neutral",
 content="(Paste legal case text here)"
)

Ideal for LLM pipelines in RAG systems, financial copilots, or multi-client dashboards.

5. Test Your Prompts Like a Researcher:

I used to put prompts straight into my products and hope for the best.

Now, I compare different versions (A/B Tests). I keep track of prompt changes. I check how good the results are.

And yeah, I work with these tools:

  • LangSmith — keeps track of prompt chains and logs responses
  • Ragas — checks relevance, hallucination, and groundedness (works great for RAG)
  • PromptLayer — handles version control + OpenAI integration

Here’s how I log one with LangSmith:

from langsmith.client import Client
client = Client()
client.create_run(
 name="earnings_summary_prompt_v1",
 inputs={"prompt": prompt_text},
 outputs={"response": model_output},
 tags=["financial", "v1", "LLM evaluation"]
)

Pro Tip: Store prompt metadata, token usage, and user ratings. Iterate monthly.

Bonus Prompt: Output in JSON Format

For devs building LLM APIs or chaining outputs to other systems:

json_prompt = PromptTemplate(
 input_variables=["query"],
 template="""
You're a data extraction assistant.
Given the input below, respond in this JSON format:
{{
 "summary": "…",
 "confidence": "high/medium/low"
}}
Input:
{query}
"""
)
json_prompt.format(query="Summarize the customer feedback and rate confidence.")

Final Thoughts: Prompting is not magic. This is UX.

You’re not just typing words into a black box. You’re designing a conversation. Shaping a user journey. Crafting an interaction model. Prompt engineering transforms basic queries into intelligent, high-impact exchanges when done intentionally.
